# Goal Builder — 六要素目标卡

把任何模糊目标压缩成一张可执行、可验收、带刹车的目标卡。核心信念：**大多数 prompt 只写"要什么"，从不写"什么时候停"。**

## 流程

1. **读原始目标**，找出其中的虚词（全部/彻底/优化/搞好/提升/improve/尽量）。每个虚词都是一个未回答的问题。
2. **能推断的自己推断**，从上下文、文件、历史对话补齐；只有影响方向的分歧才问用户，一次最多问 2 个问题。
3. **输出目标卡**，严格六段，格式如下。

## 输出格式

```
## 🎯 目标卡：<短标题>

**Objective**：<一句话，动词开头，不含任何虚词。"盖一栋三层小楼"，不是"把地搞好看点">
**Scope**：<只动什么，明确不动什么。至少一条"不许碰"的红线>
**Constraints**：<机械可识别的硬约束，逐条列出。"不许用劣质水泥"级别的具体>
**Done when**：<验收清单，每条引用具体文件、命令或可观察结果。能跑命令验证的优先>
**Stop if**：<至少 2 条停止条件：意外情况停 + 超预算停>
**Token Budget**：<资源上限：轮次/时间/token/金额，超了先停下汇报>
```

## 硬性断言（自检，不合格重写）

- Objective 恰好一句话，且不含虚词表中任何词
- Scope 含至少一条否定式边界（"不动/不许/不含"）
- Done when 每条都可被第三方客观验证（引用文件名、命令或数字）
- Stop if ≥ 2 条
- 全卡 ≤ 300 字（中文计）

## 边界

- 不替用户做价值判断（该不该做这件事），只结构化怎么做。
- 目标本身信息不足以填出 Done when 时，明说"验收标准缺失"，把缺口列成问题返给用户，不硬编。
