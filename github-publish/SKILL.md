# GitHub Publish — 零经验一键发布

把"新加git+commit+建仓+push"压成一条命令。用户永远不需要懂 git。

## 流程

1. **确认三件事**：发哪个文件夹、仓库名（默认=文件夹名）、公开还是私有（**默认私有**；公开=全世界可见，须用户明说）。
2. **执行**：`bash ~/.claude/skills/github-publish/publish.sh <文件夹> [仓库名] [--public]`
3. **报结果**：把脚本输出的仓库链接给用户；有 ❌ 就把原因翻译成人话再给下一步。

## 脚本的兜底行为（幂等，重复跑安全）

- 没登录 → 明确报错并给出登录命令，**绝不做一半**
- git 身份缺失 → 自动用 GitHub 账号配置（noreply 邮箱，不暴露真实邮箱）
- 没 .git → 自动 init；没 .gitignore → 自动生成（忽略 .DS_Store、备份文件）
- 没变更 → 提示"没有新变更"，照样确保远端同步
- 仓库不存在 → 自动创建；已存在 → 直接推送

## 硬性断言（自检）

- 从不推送含 `.bak-`、`.DS_Store` 的文件
- 新建仓库默认私有；`--public` 必须来自用户明确要求
- 任何失败都停在失败点并输出人话解释，不静默跳过

## 边界

- 一次只管一个文件夹→一个仓库的映射。
- 远端有本地没有的提交（协作场景）→ 停下来问，不自动强推。
