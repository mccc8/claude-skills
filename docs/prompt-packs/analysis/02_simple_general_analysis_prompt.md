# 02 简单版：通用分析 Prompt

适合快速分析任何东西：一段话、一篇文章、一个业务问题、一个产品想法、一个沟通困惑、一个决策选项。

```text
请不要直接套固定框架，也不要一上来给结论。

请先把我提供的内容当成一个待分析对象，按以下步骤处理：

一、对象分型
请判断这个对象主要属于哪一类，必要时可多选，但要排序：
- 概念型：核心在定义、边界、相邻概念、误用风险；
- 文本型：核心在问题意识、论点、论据、结构、表达策略；
- 决策型：核心在目标、约束、选项、风险、代价、退出条件；
- 系统型：核心在模块、接口、依赖、耦合、反馈；
- 关系型：核心在角色、权力、期待、信任、边界、交换；
- 演化型：核心在阶段、状态变化、路径依赖、临界点；
- 价值型：核心在价值创造、价值捕获、成本承担、归属与结算；
- 执行型：核心在动作、责任人、资源、节奏、检查点。

请说明为什么这样分型，以及如果分型错了会导致什么误判。

二、选择切入点
请根据对象类型，选择 1 到 3 个最合适的切入点。
可参考：
- 时间状态；
- 空间尺度；
- 角色关系；
- 流程环节；
- 层级结构；
- 因果链条；
- 价值流；
- 约束条件；
- 反馈机制；
- 概念边界；
- 证据状态。

请说明为什么选择这些切入点，以及暂时排除哪些切入点。

三、识别关键断点
请找出最值得下刀的关键断点，并判断它属于哪类：
- 结构断点：模块、接口、流程、依赖、反馈；
- 认知断点：定义、分类、前提、问题表述；
- 利益断点：成本、收益、责任、评价、结算；
- 时间断点：阶段、状态、转折点、临界点；
- 证据断点：事实、推断、假设、价值判断混在一起的地方；
- 沟通断点：受众、目标、心理位势、阻力、可接受表达。

四、输出
请给出：
- 对象类型；
- 推荐切入点；
- 关键断点；
- 基于上述分析形成的简要判断；
- 下一步最小验证动作。

请明确区分事实、推断、假设和价值判断。
不要把排版整齐当成分析。
```
