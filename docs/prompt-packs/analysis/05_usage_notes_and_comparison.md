# 05 使用建议与麦肯锡式 Prompt 对比

## 使用顺序

### 1. 早期探索
优先使用：
- `02_simple_general_analysis_prompt.md`
- 或 `03_standard_general_analysis_prompt.md`

目标是找对象类型、切入点、断点和待验证问题。

### 2. 系统 / 业务 / 组织 / 产品问题
优先使用：
- `03_standard_general_analysis_prompt.md`
- 如果确认是系统型对象，也可使用 `04_system_analysis_legacy_prompt.md`

目标是识别关键依赖、耦合点、解耦方向。

### 3. 成熟汇报
先用新版 prompt 找骨头，再用麦肯锡式 prompt 整理表达。

## 核心分工

```text
麦肯锡式 prompt 是表达整理器。
新版 prompt 是分析路由器。
```

## 麦肯锡式 Prompt 优点

- 结论先行，表达清楚；
- 适合管理层汇报；
- 适合成熟商业问题；
- 能降低新手结构化表达门槛。

## 麦肯锡式 Prompt 问题

- 容易先套框架，后看对象；
- 容易产生 MECE 幻觉；
- 维度多，但断点弱；
- 容易输出空泛建议；
- 更适合整理已有判断，不适合发现判断从哪里来。

## 新版 Prompt 优点

- 先对象分型，再选择切法；
- 能识别关键断点，而不是只列维度；
- 能显性化依赖、耦合和证据状态；
- 适合复杂对象、早期探索、跨领域分析；
- 输出结果更容易进入验证和行动。

## 新版 Prompt 问题

- Token 成本更高；
- 对输入信息质量更敏感；
- 对使用者判断力要求更高；
- 简单问题使用标准版会过度分析。

## 最终判断

```text
如果目标是快速形成汇报结构，麦肯锡式 prompt 更快。
如果目标是发现问题真正该从哪里切，新版 prompt 更强。
如果目标是成熟商业表达，麦肯锡式 prompt 够用。
如果目标是复杂对象探索、跨领域理解、组织 / 系统 / 产品问题诊断，新版 prompt 更适合。
```

## 最佳组合

```text
先用新版 prompt 找到问题的骨头；
再用麦肯锡式 prompt 给它穿西装。
```
