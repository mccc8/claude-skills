🎯目标铸卡
goal-builder
把模糊的目标或任务描述压成六要素目标卡（Objective/Scope/Constraints/Done when/Stop if/Token Budget）。
「全部、彻底、优化、搞好、improve」 「帮我把这个需求理清楚」 「这个任务怎么定目标」 「写个goal」
⏸休会
recess
休会——把当前对话/讨论强制收敛为≤150字的"最小可续状态"，防止上下文污染。
「最小可续状态」 「先到这儿」 「休会」 「暂停一下」
🏗架构制图
c4-diagram
用C4模型（Context/Container/Component/Code四层）绘制软件系统架构信息图，输出自包含SVG文件。
「画个架构图」 「这个系统的结构画出来」 「C4图」 「上下文图/容器图/组件图」
🖊白板讲解
whiteboard-explain
把文章、论文、概念或方案转换成"白板讲解图"——手绘马克笔风格的SVG信息图，像老师现场讲课的板书。
「白板讲解图」 「画个白板图」 「把这篇文章画出来」 「图解一下这个概念」
🔍触发审计
trigger-audit
审计并重写skill的触发设计——用"Skill=Trigger+Action，从用户行为逆向设计触发"原则检查description质量。
「这个skill怎么老不触发」 「审计一下我的skills」 「为什么AI不用我写的skill」 「我装了一堆skill但感觉从来没被用过」
🚀一键发布
github-publish
一键把本地文件夹发布/同步到GitHub仓库，自动完成git init、commit、建仓、push全流程，零git经验可用。
「发到github」 「传到github」 「同步仓库」 「备份到github」
⬇️下行同步
github-sync
把自己GitHub上的所有仓库拉取/更新到本地~/GitHub/并生成可双击打开的索引页index.html。
「把我的仓库拉下来」 「同步github到本地」 「我github上都有什么」 「做个仓库索引」
🗂工作区审计
agents-workspace-audit
对由 AGENTS.md / CLAUDE.md 治理的 agent 工作区做证据化审计与评分。
「评估我的项目」 「审计 AGENTS.md」 「协议体检」 「看看哪个项目管理得好」
🧠分析分诊
analysis-thinking
通用分析思维分诊器——先对象分型（概念/文本/决策/系统/关系/演化/价值/执行八型），再按需加载对应深度的分析指令，渐进式披露不全量加载。
「帮我分析一下」 「这个问题怎么拆」 「从哪儿入手」 「理不清头绪」
🙋求助分诊
smart-asking
技术/工程求助分诊器——先判"该问谁"（自查/问AI/问人/发公开社区），再按对象生成合格的提问稿，解决后强制沉淀。
「该问谁」 「这个问题该问谁」 「帮我写个提问/issue/求助帖」 「怎么问才专业」
🧭思维路由
thinking-router
思维工具路由器（untools.co 25个工具的选择器）——先诊断你卡在什么状态（迷茫/笃定/纠结/过载），再配缩放方向（微距/广角/变焦），命中一个思维工具做苏格拉底式引导。
「我卡住了」 「一团乱不知道从哪开始」 「帮我想想这事」 「我该用什么框架想这个问题」
