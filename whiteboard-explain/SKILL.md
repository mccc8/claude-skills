# Whiteboard Explain — 白板讲解图

把核心内容转换成白板讲解图：手绘风格的流程图、示意图、公式、中文注释，展示主要思路和结构，像老师现场讲课的真实板书。

## 流程

1. **萃取主线**：读原文，提炼 主题 → 3-6 个要点 → 要点间的推进/因果关系。板书讲的是"思路怎么走"，不是目录复制。
2. **绘制 SVG**（视觉规范见 [style.md](style.md)）：
   - 三色马克笔：黑色主体、红色重点、蓝色补充
   - 手绘感：轻微旋转的文字/框、波浪下划线、圈注、箭头标注、允许"涂改"痕迹（划掉的词旁边写修正）
   - 16:9 画幅，干净白底，不署名，不加水印
3. **拆图规则**：不能因空间紧凑牺牲质量。一张放不下就拆，**最多三张**，每张一个完整子主题。
4. 写入 `.svg` 文件（默认 `whiteboard-<主题>-1.svg` 起），交付时附每张图一句话导读。

## 硬性断言（自检，不过重画）

- 红色元素 ≤ 全部元素的 20%（重点太多等于没有重点）
- 每张图有一条可视的"讲解动线"（编号①②③ 或箭头链），观者知道从哪看起
- 文字总量 ≤ max(200字, 原文8%)：板书是压缩，不是誊抄
- 无成段落的长句；单个文本块视觉宽 ≤16（CJK记1、西文字符记0.5），长意思拆成"短语 + 箭头"
- 白底、16:9、无外部资源依赖

## 边界

- 软件系统架构 → 交给 c4-diagram。
- 原文观点之间没有结构（纯清单）时，明说"此文更适合列表不适合板书"，给列表替代。
