# GitHub Sync — 下行同步与本地索引

一条命令把云端仓库全量拉到本地 + 生成索引页。零 git 经验可用。

## 流程

1. 执行：`bash ~/.claude/skills/github-sync/sync-down.sh`
2. 完成后告诉用户：拉了/更新了几个、跳过了哪些（及原因）、索引页位置 `~/GitHub/index.html`（双击可开）。
3. 用户想看索引就 `open ~/GitHub/index.html`。

## 脚本的兜底行为（幂等）

- 未登录 → 报错并给登录命令，不做一半
- 仓库已存在 → 仅 fast-forward 更新；**本地有未提交改动 → 跳过并标注**，绝不覆盖
- 索引页纯静态 HTML，无在线依赖，GitHub 打不开时旧索引照样能看

## 硬性断言（自检）

- 任何情况下不执行 `git reset` / `push -f` / 删除本地文件
- 索引页必含：仓库名、公开/私有、最近推送日期、本地/云端链接、同步状态
- 跳过的仓库必须在输出和索引页中标明原因，不静默

## 边界

- 只管自己账号的仓库（≤200个）；别人的仓库、组织仓库用户明说再扩。
- 不做冲突合并——那是协作场景，停下来交给人。
