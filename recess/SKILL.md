# Recess — 休会协议

"休会"本质不是为了重启，而是为了**强制收敛当前认知状态**。输出是一个可以立即停止本轮讨论、且下次能无损续接的最小状态包。

## 规则

- 若无法访问全量历史，基于当前可见内容推断，**不得编造缺失信息**。
- 禁止复述内容、禁止补全缺失逻辑、禁止抽象空话。
- 总长度 ≤ 150 字（不含标题和格式符号）。

## 输出仅含以下 5 项

```
## ⏸ 休会状态包

**核心问题**：<1句话。当前真正试图解决的是什么，避免表面话题>
**思维推进**：<3-5个节点，只写"问题如何被改写"，格式 A → B。信息不足写"（不完整）">
**已排除路径**：<最多2条，方向 + 明确触发原因。无明确证据写"未确认排除路径">
**悬而未决**：<2-3条，仍不确定且影响下一步的问题>
**结束判定**：<收敛 / 发散 / 停滞，三选一，必须基于当前信息，不允许模糊表述>
```

## 硬性断言（自检）

- 恰好 5 个字段，一个不多一个不少
- 正文 ≤ 150 字
- 思维推进每个节点都是 "X → Y" 改写格式，不是内容摘要
- 结束判定是三个词之一，无修饰语

## 边界

- 输出后即停，不追加"下次我们可以……"之类的续聊引子——休会就是休会。
- 用户要详细会议纪要时不用本 skill（那是记录，不是收敛）。
