# 工具卡来源政策（用现成 vs 自写）

> 回应"能不写就别写新的，用 GitHub 评分高的"。原则认同——不重复造轮子。但要先分清"什么算轮子"。

## 一个关键区分（决定"用现成的"可不可行）

思维工具在 GitHub 上有三种形态，只有第三种能被本库直接用：

| 形态 | 例子 | 能用吗 |
|---|---|---|
| **代码/可视化工具** | fishbone 画鱼骨图、mermaid 画图 | ✗ 是画图不是引导思考，品类不对 |
| **心智模型清单** | gudo7208/mental-models（Munger 74模型一个 skill）| ✗ 是词典不是脚本，且架构是"一个大 skill"，和"路由器+卡"不兼容 |
| **对话式引导 skill** | 少数 First-Principles / OODA 引导脚本 | ✓ 能适配，就用 |

**~~已查证结论（上一轮）：品类空白~~ —— 2026-07-09 推翻，此判断错了。** 上轮只用 gh search 单渠道 + 关键词不准，就武断说"品类空白"。用户坚持"有可复用的"，WebSearch 一下就翻出 **tjboudreaux/cc-thinking-skills（661★/MIT/39 框架 + model-router + evals）**，已 fork 到 mccc8 + 本地挂载。教训：搜索要多渠道（gh search + WebSearch + 直接验证已知仓库），一次空结果不等于不存在。untools 的抽象阶梯/推理阶梯我自写的仍有价值（中文+冥想盆集成），但"没有现成的"是错的。

## 新增工具的决策树（今后一律走这个）

```
要加一个新思维工具
   │
   ├─ GitHub 搜"<工具名> skill / prompt / claude"，看有无高星【对话式引导】实现
   │     ├─ 有且优质 → 适配进卡片格式，卡顶标注 `参考实现: <repo链接>`，不假装原创
   │     └─ 有但只是代码/清单 → 当参考资料读，仍自写引导脚本，卡顶标注 `资料来源`
   │
   └─ 无 → 自写成卡；卡顶注 untools/原始出处链接
```

## 已有 26 张的处置（不推倒重来）

自写卡内容 = 公版经典方法（艾森豪威尔、OODA 等本就是公共知识）的引导脚本化，**不是发明新方法**，所以不存在"抄袭 GitHub 轮子"问题。风险只有一个：我没逐篇对照 untools 原文，细节可能有出入（每卡已留原站链接兜底）。

**提质路径**（按需，不批量）：某张卡被真实使用后觉得味道不对 → 那一张做一轮"GitHub 高星实现 + untools 原文"校准，标注来源。**用触发校准，不为校准而校准**——符合"重复即信号"。
