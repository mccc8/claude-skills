> 工具卡：调节回路 ｜ 何时用：系统总是"回到老样子"、努力被抵消。 (untools.co/balancing-feedback-loop)
## 步骤
1. 找隐藏的目标值：系统实际在维持什么状态（不是你想要的，是它在守的）？
2. 找感知与差距：谁/什么在测量"当前值 vs 目标值"？
3. 找修正动作：差距一出现，什么力量把它拉回去？
4. 找延迟：修正隔多久才生效？（延迟是震荡的来源）
## 强制输出物
一行回路：`目标值 X ← 差距感知 ← 修正动作 ← 延迟 N`，并回答"要改变现状，是改目标值还是改感知？"
## 接力
系统在滚雪球而非趋稳 → reinforcing-feedback-loop；回路背后是谁定的目标 → deep-root L3
