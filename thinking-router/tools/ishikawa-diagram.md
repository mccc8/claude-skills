> 工具卡：鱼骨图 ｜ 何时用：一个明确的坏结果，原因候选一大把，怕漏怕偏。 (untools.co/ishikawa-diagram)
## 步骤
1. 鱼头写问题（要具体："良率跌了 3%"，不是"质量不好"）。
2. 六根主骨分类列因：人 / 机 / 料 / 法 / 环 / 测。
3. 每根骨追问一层 why（骨上长刺）。
4. 圈出最可疑的 2 根刺——判据：既能解释时间线，又有数据可查。
## 强制输出物
六骨清单表 + `最可疑 2 因 + 各自的验证动作`（查什么数据/做什么对比）。要图 → whiteboard-explain。
## 接力
验证动作要数据 → da-* 系列（规划中）；可疑因跨部门 → conflict-resolution-diagram
