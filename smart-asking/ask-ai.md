> 子指令：面向 LLM/Chatbox 的一事一议提问稿。核心是"用定义换精准"——上下文要冗余（怕它猜），礼仪归零，但验证责任全在自己。

把用户的原始困惑加工成如下结构的 AI 指令（直接生成成品，不解释）：

**Objective**：一句话目标，先说要什么，再说卡在哪。开头就写"我的目标是 X；不要建议 Y 方案，因为 Z 约束"。
**已知条件**：具体版本号 / API 文档片段 / 物理公式 / 报错原文——能贴的原文全贴，不让 AI 猜。
**约束**：技术栈、性能、成本、协议等硬限制 + 负面约束（不要用什么）。
**输入数据**：日志、代码段、规格书片段（最小化：只留能复现问题的部分）。
**输出要求**：格式（表格/代码/图）+ 证据分级——要求 AI 区分【文档依据 / 推理 / 猜测】，不确定的参数必须标注而非编造。
**验证计划**（本稿灵魂，别的模板没有）：写明"我将如何检验答案"——跑什么命令、查哪份原始文档、对照什么标准。答案没有过验证计划前，只是候选。

**迭代协议**：追问两轮仍在打转（同类幻觉反复出现）→ 停，这是"该问人"的信号，回到路由表。

戒律：AI 的平和是陷阱——它不会骂你 RTFM，但会一本正经地编。验证计划就是你自带的 RTFM。
