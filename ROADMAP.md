# 数据分析 Skill 路线图

> 母题：**围绕不同决策类型，用个人数据写出一套可信的数据分析 Skill。公开方法，私藏数据。**
> 节奏：里程碑制，不按周排——完成一个碑，才开下一个碑。

## 三条底层规则（来自前期论证，不再重开辩论）

1. **Skill = 分析作业包，不是 prompt**：每个 skill 必须回答——辅助哪类决策、依赖什么数据口径、如何验证输出价值。答不齐三问的只是 prompt 收藏。
2. **两层分离**：方法框架、报告模板、eval 设计 → 公开（本仓库）；真实数据、字段字典、个人隐私 → 私有（本地/私仓，展示只用脱敏样例）。
3. **坏数据 eval 是灵魂**：每个 skill ≥3 个用例（正常/异常/坏数据），坏数据用例必须让 skill 承认"不足以下结论"而不是硬编。

## 资产包标准结构（每个 DA skill 一律如此）

```
<skill-name>/
  SKILL.md                # 给 Claude Code / Codex 用（Trigger+Action）
  prompt.md               # 给普通 AI 工具复制用
  schema_requirements.md  # 需要哪些字段、定义、单位（模板公开，实例私有）
  report_template.md      # 输出格式
  eval/                   # ≥3 用例：正常 / 异常 / 坏数据
  changelog.md            # 版本维护
```

## 数据源梯队（个人数据优先）

| 梯队 | 数据 | 特点 | 服务的决策类型 |
|---|---|---|---|
| 首发 | flomo 笔记导出 | 已在手（zip 导出打通），文本+时间戳 | 描述、诊断 |
| 第二 | 微信读书（readneo 经验在先） | 阅读时长/划线/书目，schema 半现成 | 描述、监控 |
| 第三 | 屏幕时间 / 健康数据 | 天然时间序列，持续产生 | 监控、预测 |
| 后备 | 开放数据（ECMWF/Kaggle） | 当个人数据撑不起某决策类型时补位 | 预测、因果 |

## 里程碑（按决策类型递进）

### M0 · 开张（当天完成）
仓库公开、README 技能表、Pages 展示页、bio、本路线图上线。
**验收**：展示页可访问，README 含 8 个既有 skill。

### M1 · 描述型 I：先让数据开口
写 `da-dataset-intake`（数据集体检：行列/类型/缺失/时间跨度/一句话画像）+ `da-dictionary-builder`（字段字典草案生成，**每个字段解释必须标注"确定 / 推测 / 不知道"——不许把猜测写成真理**）。
**第一刀**：对自己的 flomo 导出跑一次，产出 `data_profile.md` + 字典草案。
**验收**：两个 skill 各过 3 eval；脱敏样例上展示页。

### M2 · 描述型 II：从数字到决策简报
写 `da-quality-report`（缺失/重复/异常/类型检查）+ `da-eda-brief`（EDA→决策简报：结论/证据/风险/下一步）。
**验收**：产出一篇可公开的《我的笔记数据真相》分析文章——**北极星事件#1**（外部反馈以此计量）。

### M3 · 监控型：让 loop 替你盯着
写 `da-trend-summary` + `da-anomaly-triage`（异常识别与分级）。接入已有 loop 基建：笔记频率/阅读时长周报自动生成。
**验收**：连续 3 期周报无人工干预产出，异常至少被正确分级一次。

### M4 · 诊断型：为什么发生
写 `da-root-cause-candidate`（候选原因分析，明确"不冒充因果证明"）。场景：笔记断更归因、阅读下滑归因。
**验收**：坏数据 eval 通过（缺字段时列出"需补采数据"而非编故事）。

### M5 · 预测型 baseline：会预测，更会说不确定
写 `da-baseline-forecast`（时间序列 baseline + 不确定性区间；屏幕时间/阅读时长）。
**验收**：预测输出必带区间和"何时不该信"；对比 naive baseline 有提升才算过。

### M6 · 决策与评估：闭环
写 `da-decision-brief`（分析→行动简报：可选动作/成本/风险；**强制 default action 前置——看数据之前先写默认动作和"什么证据会让我改变主意"**）+ `da-skill-eval`（skill 评测器：裸 prompt vs skill vs skill+loop 对比时间/返工/结构命中）。
决策过程按 decision scribe 方式留档（当时可得数据/假设/推理），归档进 WorkDeck 产物区，防结果偏差。
**验收**：用 da-skill-eval 回评 M1-M5 全部 skill，出总榜；不合格的回炉。

## 迭代与应用机制（写完不是结束）

- **每周体检**：已有的 weekly-skill-audit loop 自动审全部 skill 触发质量。
- **价值判定七条**（每个 skill 立项前过筛，满足 ≥5 才动笔）：稳定数据源 / 字段字典 / 高频问题 / 明确输出对象 / 报告或决策用途 / eval case / 有人维护版本。
- **影响力闭环**：每完成一个里程碑 → 展示页更新 + 发一篇公开产出；北极星指标 = 外部真实使用与反馈（star、引用、转发、issue），不数虚荣指标。
- **Stop if**：同一 skill 3 轮 eval 不过 → 停下补 schema，不硬写；连续两个月里程碑无进展 → 砍范围重定义母题。

## 先不碰（明确负清单）

MLOps · 深度学习 · R 语言选型 · 大规模 Skill Hub · 商业化 · 复杂 KPI。工具默认 Python + pandas + 本地文件 + Git。
**在制上限**：同时维护的未完成 skill ≤ 3 个——先别做平台，先让三个 skill 活下来。

## 理论锚点（详见 [数据分析思考来源](docs/context/02-数据分析思考来源.md)）

- **None-One-Many**（按决策数量分类）：None 找灵感 → M1-M3；One 高风险推断 → M4/M6；Many 自动化 → M3/M5。
- **分析金律**：只对看到的数据下结论——坏数据 eval 的理论依据。
- **Default action**：看数据前先承诺默认动作与"改变主意的证据"——M6 强制项。
- **Danger Zone 自检**（Conway）：hacking+domain−statistics = 把巧合当因果；进 M4 之前必须补统计推断。
- 仓库定位与演进（Registry → Hub → Pack Manager）见 [仓库起源与定位](docs/context/01-仓库起源与定位.md)；同步规则见 [双向同步机制](docs/同步机制.md)。
